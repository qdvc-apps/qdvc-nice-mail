## 1. User request

Please help me write a Python 3 GTK3 app called "QDVC Nice Mail". Its purpose is to help the user write emails faster by offering common email components for copy-pasting.

It has three tabs:

1. Emoji: a table view of emoji. Each row has the emoji symbol and its name. Clicking on a row and then using the keyboard shortcut Ctrl+C will copy that emoji.
2. Phrases: a list of common phrases, derived from a user-provided list.
3. Signature: assembles the user's mail signature (plaintext).

Subtleties:

* The tab bar should appear ABOVE the toolbar (but still below the menubar), as each tab has different toolbar items.
* In the menubar there is an option to open a workspace folder. In the workspace folder there is a favourite_emoji.csv, phrases.csv, a mailsigs folder,
* When the emoji tab is active, the toolbar has a dropdown menu for which block of emoji is visible, by default "Favourites" with a single pre-configured favourite: 😊. The other option for the dropdown menu is "All Emoji" which is the full list of all emoji in unicode.
* The app should assign each emoji a unique ID, snake_case as the naming convention.
* When the user right-clicks on an emoji in the emoji tab, they can add it to favourites. It then gets added to the favourites list. The backend data storage for this is favourite_emoji.csv from the workspace folder, where such favourited emoji are identified by their unique ID as per the above.
* The emoji tab also has a dropdown menu for setting skin tone.
* The emoji tab also has a text search based on emoji name or description.
* The phrases tab obtains phrases from phrases.csv, and also has a UI for adding, editing and deleting phrases.

The mailsig tab generates a mailsig in the following format:

```
Kind regards,

John Smith

—

John Smith
Specialist and Superhero
Data by day, defeating villains by night

Disclaimer: a disclaimer text goes here

Message ref. YyM4mRnjHQ
```

It is generated like so:

* Everything before the m-dash comes from `mailsigs/signoff.txt`
* The block with "John Smith / Specialist and Superhero / Data by day, defeating villains by night" is one of multiple profiles. The profiles are from mailsigs/profiles/*.txt - the user can select which profile using a dropdown in the toolbar.
* The disclaimer is sourced from `mailsigs/disclaimer.txt`. The toolbar should have a toggle button for disclaimer to be included or not.
* The message ref is always displayed. The unique message ref is a 10-character string randomly generated from the following characters: "346789ABCDEFGHJKLMNPQRTUVWXYabcdefghijkmnpqrtwxyz". There should be a refresh button in the toolbar to generate a message ref.

This GTK3 app follows https://github.com/qdvc-apps/qdvc-python-gtk-app-specification but with the exception that there is no GTK4 implementation at this stage.
