* The app should assign each emoji a unique ID, snake_case as the naming convention.
* When the user right-clicks on an emoji in the emoji tab, they can add it to favourites. It then gets added to the favourites list. The backend data storage for this is favourite_emoji.csv from the workspace folder, where such favourited emoji are identified by their unique ID as per the above.
* The emoji tab also has a dropdown menu for setting skin tone.
* The emoji tab also has a text search based on emoji name or description.
* The phrases tab obtains phrases from phrases.csv, and also has a UI for adding, editing and deleting phrases.

```
Kind regards,

John Smith

—

John Smith
Specialist and Superhero
Data by day, defeating villains by night

Disclaimer: a disclaimer text goes here

Message ref. YyM4mRnjHQ
```

It is generated like so:

* Everything before the m-dash comes from `mailsigs/signoff.txt`
* The block with "John Smith / Specialist and Superhero / Data by day, defeating villains by night" is one of multiple profiles. The profiles are from mailsigs/profiles/*.txt - the user can select which profile using a dropdown in the toolbar.
* The disclaimer is sourced from `mailsigs/disclaimer.txt`. The toolbar should have a toggle button for disclaimer to be included or not.
* The message ref is always displayed. The unique message ref is a 10-character string randomly generated from the following characters: "346789ABCDEFGHJKLMNPQRTUVWXYabcdefghijkmnpqrtwxyz". There should be a refresh button in the toolbar to generate a message ref.

This GTK3 app follows https://github.com/qdvc-apps/qdvc-python-gtk-app-specification but with the exception that there is no GTK4 implementation at this stage.

1. **Menubar left padding** — every item now has a fixed-width (16px) icon slot in its child box, so all labels align to the same x-offset whether or not the item has an icon. That removes the uneven indentation.

2. **Missing shortcuts on Copy/Refresh** — the custom `Box(Image + Label)` child never displayed accelerators. `_menu_item` now takes an `accel=(keyval, mods)` pair, attaches the accelerator *and* renders it right-aligned via `Gtk.accelerator_get_label`. I routed every accelerated item (Open, Quit, Copy, Preferences, the three View tabs, Refresh) through this path, so they all show their shortcut consistently now — not just the two new ones.

3. **About icon** — the previous fallback stopped at `help-browser` and, if that was also absent, returned the missing name (blank/broken). The chain is now `help-about → help-browser → help-contents → dialog-information`; the last is part of the freedesktop standard icon set and present on essentially every theme, so an icon will show without any custom icon set. If somehow none resolve, the slot stays blank rather than rendering a broken-image placeholder.

4. **Preferences crash** — `PreferencesDialog` subclasses `Gtk.Dialog` (→ `Gtk.Window`), which has a read-only `window` GObject field; assigning `self.window` raised `RuntimeError: field is not writable`. Renamed the attribute to `self.main_window`.

Same caveat as before: I can't launch GTK3 in this environment, so items 1–3 are verified by code inspection and the reasoning above rather than by seeing them render. Item 4 (the crash) is a definite fix — that was a hard attribute collision. When you run it, the things to eyeball are the label alignment, the shortcut text on the right of each menu row, and the About icon.
